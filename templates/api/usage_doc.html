Favourites API Documentation

1. List User Favourites

GET /favourites/

Fetches all favourite hotels for the authenticated user.

Authorization: Bearer <access_token>

2. Add Favourite

POST /favourites/

Request Body:

{ "hotel": 1 }

3. Remove Favourite (Query Parameter)

DELETE /favourites/?hotel=1

4. Remove Favourite (Request Body)

DELETE /favourites/remove/

Request Body:

{ "hotel": 1 }

5. Hotel Search with Adult Number Filter

GET /hotels/search/?name=HotelName&location=City&adult_number=10

Returns hotels that can accommodate the specified number of adults.

6. Authentication

All endpoints require an Authorization token in the header:

Authorization: Bearer <access_token>

7. Error Responses

400: Invalid data or missing fields
401: Authentication credentials missing or invalid
404: Resource not found
500: Internal server error

Notifications API

This endpoint allows authenticated users to view their notifications and mark them as read.

List Notifications

Method: GET

URL: /api/notifications/

Returns a list of notifications for the authenticated user.

Example Response:

{
        "id": 1,
        "sender": 2,
        "user": 1,
        "message": "Your order has been shipped.",
        "send_time": "2025-03-21T12:34:56Z",
        "status": "0",
        "notification_type": "0",
        "is_active": true,
        "action_url": "http://example.com/order/123"
    }

Mark Notification as Read

Method: POST

URL: /api/notifications/{'{id}'}/mark_as_read/

This action marks a specific notification as read.

Example Request:

{
        "id": 1
    }

Example Response:

{
        "status": "notification marked as read"
    }

Payments, Bookings & Payment Methods API

This documentation describes the endpoints available for managing payments, bookings, and hotel payment methods. All endpoints require an authenticated user.

Payment Endpoints

List Payments

Method: GET

URL: /api/payments/

Returns a list of payments for the authenticated user.

Example Response:

{
      "booking": 5,
      "payment_method": 2,
      "transfer_image": "http://example.com/media/transfer_image.jpg",
      "payment_status": 0,
      "payment_date": "2025-03-21T12:34:56Z",
      "payment_subtotal": "100.00",
      "payment_totalamount": "110.00",
      "payment_currency": "USD",
      "payment_type": "card",
      "payment_note": "Payment for booking #5",
      "payment_discount": "0"
    }

Make Payment

Method: POST

URL: /api/payments/make_payment/

This endpoint creates a new payment for a booking.

Required Fields:

  • booking_id
  • payment_method_id
  • payment_subtotal
  • payment_totalamount
  • payment_currency
  • payment_type

Example Request:

{
      "booking_id": 5,
      "payment_method_id": 2,
      "payment_subtotal": "100.00",
      "payment_totalamount": "110.00",
      "payment_currency": "USD",
      "payment_type": "card",
      "transfer_image": "",
      "payment_note": "Payment for booking #5",
      "payment_discount": 0
    }

Example Response:

{
      "booking": 5,
      "payment_method": 2,
      "transfer_image": "http://example.com/media/transfer_image.jpg",
      "payment_status": 0,
      "payment_date": "2025-03-21T12:34:56Z",
      "payment_subtotal": "100.00",
      "payment_totalamount": "110.00",
      "payment_currency": "USD",
      "payment_type": "card",

    }

Booking Endpoints

List Bookings

Method: GET

URL: /api/bookings/

Returns a list of bookings for the authenticated user.

Example Response:

{
      "hotel": 1,
      "user": 3,
      "room": 4,
      "check_in_date": "2025-04-01",
      "check_out_date": "2025-04-05",
      "amount": "500.00",
      "status": "confirmed",
      "rooms_booked": 1
    }

Create Booking

Method: POST

URL: /api/bookings/create_booking/

This endpoint creates a new booking for the authenticated user.

Required Fields:

  • hotel_id
  • room_id
  • check_in_date
  • check_out_date
  • amount

Example Request:

{
      "hotel_id": 1,
      "room_id": 4,
      "check_in_date": "2025-04-01",
      "check_out_date": "2025-04-05",
      "amount": "500.00",
      "rooms_booked": 1
    }

Example Response:

{
      "hotel": 1,
      "user": 3,
      "room": 4,
      "check_in_date": "2025-04-01",
      "check_out_date": "2025-04-05",
